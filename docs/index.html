The house is the product surface.

          
Orbit is organized around a simple promise: the repository is alive, the work is inspectable, and money movement is controlled.

        

        

          

            Front door
            
Read before acting

            
Orbit can list and inspect public issues, including comments, before deciding whether work is safe and useful.

          

          

            Budget aware
            
Stop before waste

            
Daily and monthly AI-food budgets are checked before expensive thinking continues, with local fallback available.

          

          

            Proof first
            
Public cycle trail

            
Each cycle can record what Orbit saw, what it changed, what it refused, and what still needs owner review.

          

          

            Private treasury
            
Config stays private

            
Reward routing is described as a guarded configuration path instead of publishing route percentages or payout asset details.

          

          

            Identity
            
One product, many members

            
Orbit presents one public identity while internal members handle attention, memory, treasury, care, safety, and proof.

          

          

            Risk guard
            
Hostile input checks

            
Scam phrasing, wallet drains, unknown recipients, and unsafe financial links are scanned before any sensitive action.

          

          

            Launch prep
            
Native token, gated

            
Token launch and reward claim paths stay dry-run until explicit environment flags and private signer settings exist.

          

        

      


      

        

          
Financial actions are blocked by default.

          
Orbit can reason about treasury operations, but public issue content cannot silently redirect funds or bypass approval gates.
